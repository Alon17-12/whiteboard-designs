סקירה כללית

מהו WHITEBOARD Live?

WHITEBOARD Live הוא מודול וידאו-שיעור חי המוטמע ישירות בתוך פלטפורמת ה-LMS של WHITEBOARD. בניגוד לפתרונות חיצוניים כמו Zoom או Google Meet, השיעור החי מתקיים בתוך הממשק עצמו — כולל כלים אינטראקטיביים כמו סקרים, חידונים, משחקים וטיימרים שפועלים ישירות על הווידאו.

קונספט ליבה

חוויית Zoom-like עם וידאו חי, שיתוף מסך וצ'אט — אך עם שכבה אינטראקטיבית מלאה:

  • סקרים בזמן אמת — המרצה מפעיל סקר, התלמידים מצביעים, התוצאות מופיעות בשידור חי
  • חידונים עם ניקוד — שאלות עם טיימר, ניקוד מצטבר, לוח תוצאות
  • משחקים — זיכרון, טריוויה ועוד — משולבים בתוך השיעור
  • לוח לבן משותף — ציור וכתיבה בזמן אמת
  • חדרי דיון (Breakout Rooms) — חלוקה לקבוצות קטנות
  • הקלטה אוטומטית — כולל תמלול AI ותקציר

בחירת טכנולוגיה: LiveKit

לאחר בחינת חלופות (Twilio, Agora, Daily.co, Vonage) הבחירה נפלה על LiveKit — פלטפורמת WebRTC קוד פתוח.

למה LiveKit?
קוד פתוח — אין vendor lock-in, אפשר לארח עצמאית
ללא עלות לפי דקה — חיסכון משמעותי לעומת Twilio/Agora
SFU מובנה — ארכיטקטורת Selective Forwarding Unit לביצועים מעולים
Data Channels — תמיכה מובנית בערוצי נתונים לאינטראקציות
Egress/Ingress — הקלטה ושידור מובנים
AI Agent Framework — תשתית מוכנה לשילוב בינה מלאכותית
SDKs — תמיכה ב-JavaScript, React, Swift, Kotlin, Flutter ועוד

TURN Server

נדרש כדי לעקוף NAT ו-firewalls. LiveKit כולל TURN מובנה (מומלץ), או אפשר להשתמש ב-coturn חיצוני.

ה-TURN המובנה של LiveKit מספיק ברוב המקרים. coturn חיצוני נדרש רק בסביבות עם firewalls קשיחים במיוחד.

אחסון — S3 / GCS

הקלטות נשמרות ב-S3 (AWS) או Google Cloud Storage. יש להגדיר bucket ייעודי עם signed URLs לגישה מאובטחת. 

// S3 configuration for Egress
{
  "s3": {
    "access_key": "AKIA...",
    "secret": "...",
    "region": "eu-west-1",
    "bucket": "whiteboard-recordings"
  }
}

Backend API — Node.js

שרת Node.js שמנהל את הלוגיקה העסקית: יצירת חדרים, הנפקת tokens, ניהול הקלטות ואינטראקציות.

מסד נתונים — PostgreSQL

5 טבלאות עיקריות נדרשות:

-- 1. סשנים חיים
CREATE TABLE live_sessions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id     UUID REFERENCES courses(id),
  title         VARCHAR(255) NOT NULL,
  scheduled_at  TIMESTAMPTZ,
  started_at    TIMESTAMPTZ,
  ended_at      TIMESTAMPTZ,
  duration      INTEGER,          -- planned duration in minutes
  status        VARCHAR(20) DEFAULT 'scheduled',  -- scheduled/live/ended/cancelled
  preset_id     UUID REFERENCES live_presets(id),
  recording_url TEXT,
  livekit_room  VARCHAR(100),
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now()
);

-- 2. משתתפים
CREATE TABLE live_participants (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id   UUID REFERENCES live_sessions(id),
  user_id      UUID REFERENCES users(id),
  role         VARCHAR(20) NOT NULL,   -- teacher/student/assistant
  joined_at    TIMESTAMPTZ,
  left_at      TIMESTAMPTZ,
  duration_sec INTEGER
);

-- 3. אינטראקציות
CREATE TABLE live_interactions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  UUID REFERENCES live_sessions(id),
  type        VARCHAR(30) NOT NULL,   -- poll/quiz/game/timer/whiteboard/breakout
  data_json   JSONB NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- 4. פריסטים
CREATE TABLE live_presets (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id),
  name        VARCHAR(100) NOT NULL,
  description TEXT,
  items_json  JSONB NOT NULL,     -- ordered array of interaction configs
  is_template BOOLEAN DEFAULT false,
  created_at  TIMESTAMPTZ DEFAULT now(),
  updated_at  TIMESTAMPTZ DEFAULT now()
);

-- 5. הקלטות
CREATE TABLE live_recordings (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  UUID REFERENCES live_sessions(id),
  url         TEXT NOT NULL,
  duration    INTEGER,             -- seconds
  size_bytes  BIGINT,
  format      VARCHAR(10) DEFAULT 'mp4',
  transcript  TEXT,                -- AI-generated transcript
  summary     TEXT,                -- AI-generated summary
  created_at  TIMESTAMPTZ DEFAULT now()
);

מסכי המערכת (UI Screens)

להלן כל המסכים שעוצבו עבור מערכת ה-Live, כולל תיאור הפונקציונליות והדרישות מצד ה-Backend עבור כל מסך.

לובי (Pre-Join)
מסך הכנה לפני כניסה לשיעור — בדיקת מצלמה ומיקרופון, בחירת מכשירים, תצוגה מקדימה של הווידאו, שם המשתמש.
Backend: endpoint לאימות token, בדיקת הרשאות (האם הסשן פעיל, האם המשתמש רשום), החזרת room token מ-LiveKit.
פתח מסך עיצוב ←
חדר מרצה
המסך המרכזי של המרצה — במה עם וידאו, גלריית משתתפים, סרגל כלים תחתון (מיקרופון, מצלמה, שיתוף מסך, הקלטה), סיידבר עם 3 טאבים (משתתפים / צ'אט / כלים).
Backend: room management API, mute/kick participants, start/stop recording, publish data channel messages (interactions), webhook לעדכון סטטוס.
פתח מסך עיצוב ←
חדר תלמיד
מסך מפושט עם תצוגת מרצה מוגדלת, צ'אט, הרמת יד, ריאקציות. המרצה בוידאו גדול, סרגל כלים מצומצם.
Backend: token עם הרשאות מוגבלות (canPublish: false כברירת מחדל), API להרמת יד, listen ל-data channel עבור אינטראקציות.
פתח מסך עיצוב ←
אינטראקציות על הלייב
overlay של סקר, חידון, משחק, טיימר, לוח לבן — מופעלים מטאב הכלים בחדר המרצה. מוצגים כ-overlay על גבי הווידאו אצל כל המשתתפים.
Backend: שמירת תוצאות אינטראקציות ב-DB, ניהול state של אינטראקציה פעילה, אגרגציית תשובות.
סקר ושאלה  |  משחק  |  תצוגת תלמיד
חדרי דיון (Breakout Rooms)
overlay עם glassmorphism לחלוקת תלמידים לחדרים קטנים. המרצה יכול לקבוע חלוקה ידנית או אקראית, להגדיר זמן, ולעבור בין חדרים.
Backend: יצירת חדרי LiveKit נפרדים (sub-rooms), העברת participants בין rooms, טיימר לסגירה אוטומטית, broadcast הודעות לכל החדרים.
פתח מסך עיצוב ←
הקלטות
רשימת הקלטות עם נגן מובנה, תמלול BOARDi AI, אפשרות הורדה ושיתוף. חיפוש בתוך התמלול, פרקים אוטומטיים.
Backend: API לרשימת הקלטות, signed URL generation, שמירת תמלול, webhook מ-Egress בסיום הקלטה.
פתח מסך עיצוב ←
תזמון שיעור חי
טופס 2 עמודות: פרטי שיעור (שם, תאריך, שעה, משך) + בחירת פריסט, הזמנת תלמידים, popup סיכום לפני שליחה.
Backend: CRUD ל-live_sessions, שליחת הזמנות (email/notification), יצירת calendar event, cron job לתזכורות (15 דקות לפני).
פתח מסך עיצוב ←
ספריית פריסטים
ספרייה של תבניות מוכנות לשיעורים חיים — כל פריסט הוא רשימה מסודרת של אינטראקציות. תבניות מערכת + תבניות אישיות.
Backend: CRUD ל-live_presets, seed data לתבניות מערכת (is_template: true), שכפול preset.
פתח מסך עיצוב ←
עורך פריסט
Drag & Drop builder — גרירת אינטראקציות לרשימה, עריכת כל אינטראקציה (שאלה, תשובות, זמן), שינוי סדר, תצוגה מקדימה.
Backend: שמירת items_json כ-JSONB, validation של מבנה, auto-save.
פתח מסך עיצוב ←
הגדרות לייב
סקשן בתוך הגדרות הקורס/הכשרה — הפעלת מודול Live, ברירות מחדל (הקלטה אוטומטית, צ'אט, הרשאות תלמידים), מגבלת משתתפים.
Backend: עמודות נוספות בטבלת course_settings, API לעדכון, migration.
פתח הגדרות קורס ←  |  הגדרות הכשרה ←

LiveKit Integration — שלב אחר שלב

שלבי היישום המרכזיים, עם קוד לדוגמה עבור כל שלב.

שלב 1 — התקנה

npm install livekit-client livekit-server-sdk

שלב 2 — Server: הנפקת Token

ה-Token מגדיר את זהות המשתמש והרשאותיו בחדר. המרצה מקבל canPublish: true, תלמיד מקבל false כברירת מחדל.

import { AccessToken } from 'livekit-server-sdk';

async function generateToken(userId, userName, roomName, role) {
  const token = new AccessToken(
    process.env.LIVEKIT_API_KEY,
    process.env.LIVEKIT_API_SECRET,
    {
      identity: userId,
      name: userName,
      metadata: JSON.stringify({ role }),
    }
  );

  token.addGrant({
    roomJoin: true,
    room: roomName,
    canPublish: role === 'teacher' || role === 'assistant',
    canSubscribe: true,
    canPublishData: true,  // needed for chat & interactions
  });

  return await token.toJwt();
}

שלב 5 — Subscribe לטראקים של אחרים

room.on(RoomEvent.TrackSubscribed, (track, publication, participant) => {
  const element = track.attach();

  if (track.kind === 'video') {
    element.style.width = '100%';
    document.getElementById('video-grid').appendChild(element);
  } else if (track.kind === 'audio') {
    document.body.appendChild(element); // audio elements are hidden
  }
});

room.on(RoomEvent.TrackUnsubscribed, (track) => {
  track.detach().forEach(el => el.remove());
});

room.on(RoomEvent.ParticipantDisconnected, (participant) => {
  console.log('Participant left:', participant.identity);
});

שלב 6 — Data Channels לאינטראקציות

import { DataPacket_Kind } from 'livekit-client';

// Send interaction (teacher)
const encoder = new TextEncoder();
const data = encoder.encode(JSON.stringify({
  type: 'poll_start',
  id: 'poll_abc123',
  question: 'מה השפה הראשונה שלמדתם?',
  options: ['JavaScript', 'Python', 'Java', 'C++'],
  duration: 30,
}));

await room.localParticipant.publishData(
  data,
  DataPacket_Kind.RELIABLE  // use RELIABLE for interactions
);

// Receive interaction (all participants)
room.on(RoomEvent.DataReceived, (payload, participant) => {
  const decoder = new TextDecoder();
  const message = JSON.parse(decoder.decode(payload));

  switch (message.type) {
    case 'poll_start':
      showPollOverlay(message);
      break;
    case 'quiz_start':
      showQuizOverlay(message);
      break;
    case 'chat_message':
      appendChatMessage(message);
      break;
    // ... more types
  }
});

שלב 7 — הקלטה (Egress)

import { EgressClient } from 'livekit-server-sdk';

const egressClient = new EgressClient(
  process.env.LIVEKIT_URL,
  process.env.LIVEKIT_API_KEY,
  process.env.LIVEKIT_API_SECRET
);

// Start composite recording (one video with all participants)
const egress = await egressClient.startRoomCompositeEgress(
  roomName,
  {
    file: {
      filepath: `s3://whiteboard-recordings/\${sessionId}/recording.mp4`,
      output: {
        s3: {
          accessKey: process.env.AWS_ACCESS_KEY,
          secret: process.env.AWS_SECRET_KEY,
          region: 'eu-west-1',
          bucket: 'whiteboard-recordings',
        }
      }
    }
  }
);

// Store egress ID for later stop
const egressId = egress.egressId;

// Stop recording
await egressClient.stopEgress(egressId);

פרוטוקול אינטראקציות — Data Channel Protocol

כל האינטראקציות (סקרים, חידונים, משחקים וכו') מתקשרות דרך LiveKit Data Channels. הפרוטוקול מבוסס על JSON עם שדה type שמגדיר את סוג ההודעה. כל ההודעות נשלחות כ-DataPacket_Kind.RELIABLE (TCP-like).

שימו לב: ריאקציות (emoji) נשלחות כ-LOSSY (UDP-like) כי הן לא קריטיות ויש להן volume גבוה.

Poll — סקר

// מרצה מפעיל סקר
{
  "type": "poll_start",
  "id": "poll_abc123",
  "question": "איזה framework אתם מעדיפים?",
  "options": ["React", "Vue", "Angular", "Svelte"],
  "duration": 30,
  "allowMultiple": false
}

// תלמיד מצביע
{
  "type": "poll_vote",
  "id": "poll_abc123",
  "option": 0,
  "userId": "user_456"
}

// מרצה סוגר סקר + תוצאות
{
  "type": "poll_end",
  "id": "poll_abc123",
  "results": [12, 8, 5, 3],
  "totalVotes": 28
}

Quiz — חידון

// מרצה מפעיל שאלה
{
  "type": "quiz_start",
  "id": "q_789",

הקלטה ושידור

LiveKit Egress Service

Egress הוא שירות נפרד של LiveKit שמאפשר הקלטה ושידור (streaming). הוא רץ כ-container נפרד ומתחבר לחדר כ-"participant סמוי".

הפעלת Egress Service

docker run -d \
  --name livekit-egress \
  -e EGRESS_CONFIG_FILE=/egress.yaml \
  -v /path/to/egress.yaml:/egress.yaml \
  livekit/egress

קובץ תצורה — egress.yaml

api_key: WHITEBOARD_API_KEY
api_secret: <your-api-secret>
ws_url: wss://live.whiteboard.co.il

s3:
  access_key: AKIA...
  secret: ...
  region: eu-west-1
  bucket: whiteboard-recordings

סוגי הקלטה

סוגתיאורשימוש
Room Composite קובץ MP4 אחד עם כל המשתתפים, כולל layout אוטומטי ברירת מחדל — מתאים לרוב המקרים
Track Composite בחירת טראקים ספציפיים להקלטה הקלטת המרצה בלבד, או מסך בלבד
Per-Track (Web Egress) קובץ נפרד לכל טראק עריכה מתקדמת בפוסט-פרודקשן

Signed URLs לגישה מאובטחת

import { S3Client, GetObjectCommand } from '@aws-sdk/client-s3';
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';

async function getRecordingUrl(key) {
  const client = new S3Client({ region: 'eu-west-1' });
  const command = new GetObjectCommand({
    Bucket: 'whiteboard-recordings',
    Key: key,
  });
  return await getSignedUrl(client, command, { expiresIn: 3600 });
}

Webhook בסיום הקלטה

import { WebhookReceiver } from 'livekit-server-sdk';

const receiver = new WebhookReceiver(
  process.env.LIVEKIT_API_KEY,
  process.env.LIVEKIT_API_SECRET
);

app.post('/webhook/livekit', async (req, res) => {
  const event = await receiver.receive(req.body, req.headers.authorization);

  if (event.event === 'egress_ended') {
    const { egressId, roomName, file } = event.egressInfo;

    // Save recording to DB
    await db.query(`
      INSERT INTO live_recordings (session_id, url, duration, size_bytes)
      VALUES ($1, $2, $3, $4)
    `, [sessionId, file.filename, file.duration, file.size]);

    // Trigger AI transcription (async)
    await triggerTranscription(sessionId, file.filename);
  }

  res.sendStatus(200);
});

// Cron job — runs every minute
async function checkUpcomingReminders() {
  const sessions = await db.query(`
    SELECT * FROM live_sessions
    WHERE status = 'scheduled'
    AND scheduled_at BETWEEN NOW() AND NOW() + INTERVAL '15 minutes'
    AND reminder_sent = false
  `);

  for (const session of sessions.rows) {
    await sendReminderToParticipants(session.id);
    await db.query(`UPDATE live_sessions SET reminder_sent = true WHERE id = $1`, [session.id]);
  }
}

BOARDi AI Integration (Phase 5)

זהו שלב מתקדם (Phase 5) — יש לבנות אותו רק אחרי שה-Live הבסיסי, האינטראקציות, וההקלטה עובדים. אל תנסו לעשות הכל בבת אחת.

LiveKit AI Agent Framework

LiveKit מספקת framework מוכן לבניית AI Agents שמשתתפים בחדרים. ה-Agent מתחבר כמשתתף רגיל ויכול לשמוע, לראות ולשלוח data. 

// Python — LiveKit Agent (recommended for AI)
from livekit.agents import AutoSubscribe, JobContext, WorkerOptions, cli
from livekit.agents.transcription import STTSegmentsForwarder
from livekit.plugins import deepgram

async def entrypoint(ctx: JobContext):
    # Connect to room as AI agent
    await ctx.connect(auto_subscribe=AutoSubscribe.AUDIO_ONLY)

    # Real-time speech-to-text
    stt = deepgram.STT(language="he")  # Hebrew support

    # Forward transcription to data channel
    forwarder = STTSegmentsForwarder(
        room=ctx.room,
        participant=ctx.room.local_participant,
    )

    async for event in stt.stream():
        await forwarder.forward(event)
        # Also save to DB for later
        await save_transcript_segment(ctx.room.name, event)

if __name__ == "__main__":
    cli.run_app(WorkerOptions(entrypoint_fnc=entrypoint))

// Triggered when recording is ready
async function generateSessionSummary(sessionId) {
  // 1. Get transcript
  const transcript = await getTranscript(sessionId);

  // 2. Send to Claude for summary
  const summary = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',

      \${transcript}`
    }],
    max_tokens: 2000,
  });

  // 3. Save to DB
  await db.query(

    [summary.content[0].text, sessionId]
  );

  // 4. Notify students
  await notifyStudents(sessionId, 'session_summary_ready');
}

מפת דרכים ליישום

הפיתוח מחולק ל-6 שלבים, מהבסיסי ביותר (חדר וידאו פשוט) ועד ליכולות מתקדמות (AI). הזמנים מבוססים על צוות של 1-2 מפתחי fullstack.

שלב משך מה כולל עדיפות
Phase 1
חדר בסיסי		 3-4 שבועות וידאו/אודיו בסיסי, שיתוף מסך, חיבור לחדר (lobby + room), token management, device selection P0 — חובה
Phase 2
כלי מרצה		 2-3 שבועות השתקת כולם, הרחקת משתתף, רשימת משתתפים, צ'אט בזמן אמת, הרמת יד, ריאקציות P0 — חובה
Phase 3
אינטראקציות		 3-4 שבועות סקר, חידון, משחק, טיימר, לוח לבן, חדרי דיון, פריסטים ועורך פריסט P1 — גבוה
Phase 4
הקלטה + VOD		 2-3 שבועות Egress setup, הקלטת composite, נגן VOD, signed URLs, ניהול הקלטות, webhook P1 — גבוה
Phase 5
BOARDi AI		 2-3 שבועות תמלול בזמן אמת, סיכום אוטומטי, Q&A chatbot, הערות חכמות, פרקים P2 — בינוני

אופטימיזציה		 2-3 שבועות Adaptive bitrate, simulcast, multi-region deployment, mobile optimization, HLS לקהל גדול, monitoring P2 — בינוני
