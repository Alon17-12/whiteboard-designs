סקירה כללית

מהו WHITEBOARD Live?

WHITEBOARD Live הוא מודול וידאו-שיעור חי המוטמע ישירות בתוך פלטפורמת ה-LMS של WHITEBOARD. בניגוד לפתרונות חיצוניים כמו Zoom או Google Meet, השיעור החי מתקיים בתוך הממשק עצמו — כולל כלים אינטראקטיביים כמו סקרים, חידונים, משחקים וטיימרים שפועלים ישירות על הווידאו.

קונספט ליבה

חוויית Zoom-like עם וידאו חי, שיתוף מסך וצ'אט — אך עם שכבה אינטראקטיבית מלאה:

  • סקרים בזמן אמת — המרצה מפעיל סקר, התלמידים מצביעים, התוצאות מופיעות בשידור חי
  • חידונים עם ניקוד — שאלות עם טיימר, ניקוד מצטבר, לוח תוצאות
  • משחקים — זיכרון, טריוויה ועוד — משולבים בתוך השיעור
  • לוח לבן משותף — ציור וכתיבה בזמן אמת
  • חדרי דיון (Breakout Rooms) — חלוקה לקבוצות קטנות
  • הקלטה אוטומטית — כולל תמלול AI ותקציר

בחירת טכנולוגיה: LiveKit

לאחר בחינת חלופות (Twilio, Agora, Daily.co, Vonage) הבחירה נפלה על LiveKit — פלטפורמת WebRTC קוד פתוח.

למה LiveKit?
קוד פתוח — אין vendor lock-in, אפשר לארח עצמאית
ללא עלות לפי דקה — חיסכון משמעותי לעומת Twilio/Agora
SFU מובנה — ארכיטקטורת Selective Forwarding Unit לביצועים מעולים
Data Channels — תמיכה מובנית בערוצי נתונים לאינטראקציות
Egress/Ingress — הקלטה ושידור מובנים
AI Agent Framework — תשתית מוכנה לשילוב בינה מלאכותית
SDKs — תמיכה ב-JavaScript, React, Swift, Kotlin, Flutter ועוד

Tech Stack

כל ה-Frontend נבנה ב-React + Next.js עם TypeScript (strict mode). הארכיטקטורה כוללת:

React + Next.js — App Router, Server Components כברירת מחדל, Client Components רק עבור ממשק אינטראקטיבי (וידאו, צ'אט, כלים)
TypeScript — strict mode, ללא any, כל ה-interfaces מוגדרים מראש
Bunny CDN + Bunny Stream — אחסון מדיה, הפצת הקלטות, video playback
Trigger.dev — כל background jobs: שליחת אימיילים, עיבוד וידאו, דוחות, תזכורות מתוזמנות

TURN Server

נדרש כדי לעקוף NAT ו-firewalls. LiveKit כולל TURN מובנה (מומלץ), או אפשר להשתמש ב-coturn חיצוני.

ה-TURN המובנה של LiveKit מספיק ברוב המקרים. coturn חיצוני נדרש רק בסביבות עם firewalls קשיחים במיוחד.

אחסון — Bunny Storage + Bunny CDN

הקלטות נשמרות ב-Bunny Storage ומופצות דרך Bunny CDN. וידאו playback דרך Bunny Stream. הגישה מאובטחת באמצעות Bunny token authentication.

Backend API — Next.js API Routes

Next.js API Routes (App Router) מנהלות את כל הלוגיקה העסקית: יצירת חדרים, הנפקת tokens, ניהול הקלטות ואינטראקציות. כל ה-API routes נכתבות ב-TypeScript strict mode.

Background Jobs — Trigger.dev

כל ה-background jobs רצים דרך Trigger.dev (לא cron jobs, לא setTimeout):

  • שליחת אימיילים — הזמנות לשיעור חי, תזכורות
  • עיבוד וידאו — העלאה ל-Bunny Stream אחרי הקלטה, יצירת thumbnails
  • דוחות — סיכומי סשן, סטטיסטיקות נוכחות
  • תזכורות מתוזמנות — תזכורת 15 דקות לפני סשן (החלפת cron jobs)

מסד נתונים — PostgreSQL + Prisma ORM

משתמשים ב-Prisma ORM עם TypeScript לגישה type-safe למסד הנתונים. כל ה-queries עוברות דרך Prisma client עם auto-generated types.

5 טבלאות עיקריות נדרשות:

-- 1. סשנים חיים
CREATE TABLE live_sessions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  course_id     UUID REFERENCES courses(id),
  title         VARCHAR(255) NOT NULL,
  scheduled_at  TIMESTAMPTZ,
  started_at    TIMESTAMPTZ,
  ended_at      TIMESTAMPTZ,
  duration      INTEGER,          -- planned duration in minutes
  status        VARCHAR(20) DEFAULT 'scheduled',  -- scheduled/live/ended/cancelled
  preset_id     UUID REFERENCES live_presets(id),
  recording_url TEXT,
  livekit_room  VARCHAR(100),
  created_at    TIMESTAMPTZ DEFAULT now(),
  updated_at    TIMESTAMPTZ DEFAULT now()
);

-- 2. משתתפים
CREATE TABLE live_participants (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id   UUID REFERENCES live_sessions(id),
  user_id      UUID REFERENCES users(id),
  role         VARCHAR(20) NOT NULL,   -- teacher/student/assistant
  joined_at    TIMESTAMPTZ,
  left_at      TIMESTAMPTZ,
  duration_sec INTEGER
);

-- 3. אינטראקציות
CREATE TABLE live_interactions (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  UUID REFERENCES live_sessions(id),
  type        VARCHAR(30) NOT NULL,   -- poll/quiz/game/timer/whiteboard/breakout
  data_json   JSONB NOT NULL,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- 4. פריסטים
CREATE TABLE live_presets (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id),
  name        VARCHAR(100) NOT NULL,
  description TEXT,
  items_json  JSONB NOT NULL,     -- ordered array of interaction configs
  is_template BOOLEAN DEFAULT false,
  created_at  TIMESTAMPTZ DEFAULT now(),
  updated_at  TIMESTAMPTZ DEFAULT now()
);

-- 5. הקלטות
CREATE TABLE live_recordings (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id  UUID REFERENCES live_sessions(id),
  url         TEXT NOT NULL,
  duration    INTEGER,             -- seconds
  size_bytes  BIGINT,
  format      VARCHAR(10) DEFAULT 'mp4',
  transcript  TEXT,                -- AI-generated transcript
  summary     TEXT,                -- AI-generated summary
  created_at  TIMESTAMPTZ DEFAULT now()
);

LiveKit Integration — שלב אחר שלב

שלבי היישום המרכזיים, עם קוד לדוגמה עבור כל שלב.

שלב 1 — התקנה

npm install livekit-client livekit-server-sdk @livekit/components-react @trigger.dev/sdk

שלב 2 — Server: הנפקת Token (Next.js API Route)

ה-Token מגדיר את זהות המשתמש והרשאותיו בחדר. המרצה מקבל canPublish: true, תלמיד מקבל false כברירת מחדל.

// TypeScript
// types/live.ts
interface LiveSession {
  id: string;
  courseId: string;
  title: string;
  scheduledAt: Date;
  duration: number;
  status: 'scheduled' | 'live' | 'ended';
  presetId?: string;
}

type ParticipantRole = 'teacher' | 'student' | 'assistant';

interface TokenRequest {
  userId: string;
  userName: string;
  roomName: string;
  role: ParticipantRole;
}

// app/api/live/token/route.ts (Next.js API Route)
import { AccessToken } from 'livekit-server-sdk';
import { NextRequest, NextResponse } from 'next/server';

export async function POST(req: NextRequest): Promise<NextResponse> {
  const { userId, userName, roomName, role }: TokenRequest = await req.json();

  const token = new AccessToken(
    process.env.LIVEKIT_API_KEY!,
    process.env.LIVEKIT_API_SECRET!,
    {
      identity: userId,
      name: userName,
      metadata: JSON.stringify({ role }),
    }
  );

  token.addGrant({
    roomJoin: true,
    room: roomName,
    canPublish: role === 'teacher' || role === 'assistant',
    canSubscribe: true,
    canPublishData: true,  // needed for chat & interactions
  });

  const jwt: string = await token.toJwt();
  return NextResponse.json({ token: jwt });
}

    element.style.width = '100%';
    document.getElementById('video-grid').appendChild(element);
  } else if (track.kind === 'audio') {
    document.body.appendChild(element); // audio elements are hidden
  }
});

room.on(RoomEvent.TrackUnsubscribed, (track: RemoteTrack) => {
  track.detach().forEach((el: HTMLMediaElement) => el.remove());
});

room.on(RoomEvent.ParticipantDisconnected, (participant: RemoteParticipant) => {

פרוטוקול אינטראקציות — Data Channel Protocol

כל האינטראקציות (סקרים, חידונים, משחקים וכו') מתקשרות דרך LiveKit Data Channels. הפרוטוקול מבוסס על JSON עם שדה type שמגדיר את סוג ההודעה. כל ההודעות נשלחות כ-DataPacket_Kind.RELIABLE (TCP-like).

Bunny Token Authentication לגישה מאובטחת

במקום signed URLs של S3, משתמשים ב-Bunny token authentication. Bunny CDN מפיצה את ההקלטות, ו-Bunny Stream משמש ל-video playback.

// TypeScript
// lib/bunny.ts
import crypto from 'crypto';

interface BunnyTokenOptions {
  path: string;
  expiresIn: number; // seconds
}

function generateBunnyToken(options: BunnyTokenOptions): string {
  const expires: number = Math.floor(Date.now() / 1000) + options.expiresIn;
  const hashableBase: string = process.env.BUNNY_TOKEN_KEY! + options.path + expires;
  const token: string = crypto.createHash('sha256')
    .update(hashableBase).digest('base64')
    .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
  return `\${options.path}?token=\${token}&expires=\${expires}`;
}

async function getRecordingUrl(key: string): Promise<string> {
  const cdnBase: string = 'https://whiteboard-cdn.b-cdn.net';
  const tokenPath: string = generateBunnyToken({ path: `/\${key}`, expiresIn: 3600 });
  return `\${cdnBase}\${tokenPath}`;
}

// For video playback — use Bunny Stream embed
function getBunnyStreamUrl(videoId: string): string {
  const libraryId: string = process.env.BUNNY_STREAM_LIBRARY_ID!;
  return `https://iframe.mediadelivery.net/embed/\${libraryId}/\${videoId}`;
}

// TypeScript
// app/api/webhook/livekit/route.ts (Next.js API Route)
import { WebhookReceiver } from 'livekit-server-sdk';
import { NextRequest, NextResponse } from 'next/server';
import { processRecording } from '@/trigger/process-recording';

const receiver: WebhookReceiver = new WebhookReceiver(
  process.env.LIVEKIT_API_KEY!,
  process.env.LIVEKIT_API_SECRET!
);

export async function POST(req: NextRequest): Promise<NextResponse> {
  const body: string = await req.text();
  const auth: string = req.headers.get('authorization') ?? '';
  const event = await receiver.receive(body, auth);

  if (event.event === 'egress_ended') {
    const { egressId, roomName, file } = event.egressInfo;

    // Trigger.dev job for post-recording processing
    await processRecording.trigger({
      sessionId,
      recordingUrl: file.filename,
      duration: file.duration,
      sizeBytes: file.size,
    });
  }

  return new NextResponse(null, { status: 200 });
}

LiveKit AI Agent Framework

LiveKit מספקת framework מוכן לבניית AI Agents שמשתתפים בחדרים. ה-Agent מתחבר כמשתתף רגיל ויכול לשמוע, לראות ולשלוח data. 

// Python — LiveKit Agent (recommended for AI)
from livekit.agents import AutoSubscribe, JobContext, WorkerOptions, cli
from livekit.agents.transcription import STTSegmentsForwarder
from livekit.plugins import deepgram

async def entrypoint(ctx: JobContext):
    # Connect to room as AI agent
    await ctx.connect(auto_subscribe=AutoSubscribe.AUDIO_ONLY)

    # Real-time speech-to-text
    stt = deepgram.STT(language="he")  # Hebrew support

    # Forward transcription to data channel
    forwarder = STTSegmentsForwarder(
        room=ctx.room,
        participant=ctx.room.local_participant,
    )

    async for event in stt.stream():
        await forwarder.forward(event)
        # Also save to DB for later
        await save_transcript_segment(ctx.room.name, event)

if __name__ == "__main__":
    cli.run_app(WorkerOptions(entrypoint_fnc=entrypoint))

// TypeScript
// Triggered when recording is ready (via Trigger.dev)

interface SessionSummary {
  keyPoints: string[];
  questionsRaised: string[];
  tasks: string[];
  rawText: string;
}

async function generateSessionSummary(sessionId: string): Promise<SessionSummary> {
  // 1. Get transcript
  const transcript = await getTranscript(sessionId);

  // 2. Send to Claude for summary
  const summary = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    messages: [{
      role: 'user',
      content: `סכם את השיעור הבא בעברית. חלק ל: נקודות מפתח, שאלות שעלו, משימות.

      תמלול:
      \${transcript}`
    }],
    max_tokens: 2000,
  });

  // 3. Save to DB
  await db.query(
    `UPDATE live_recordings SET summary = $1 WHERE session_id = $2`,
    [summary.content[0].text, sessionId]
  );

  // 4. Notify students
  await notifyStudents(sessionId, 'session_summary_ready');
}
