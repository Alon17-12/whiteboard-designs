מערכת וובינרים
כאירוע השקה לגיוס לידים

האדמין יוצר אירוע וובינר ככלי לגיוס והמרת לידים. עצם היצירה פותחת אוטומטית 2 רשימות תפוצה (לידים/לקוחות), חדר צ׳אט ייעודי במועדון, ושלד סדרת מיילים מתוזמנת. הוובינר אינו מערכת עצמאית — הוא שכבת אורקסטרציה דקה מעל שלושה מודולים קיימים: מנוע הדיוור, המועדון, וסטטוס הלקוח ב‑CRM. מדריך זה מתאר את החיבור, ה‑schema, האוטומציה והצ׳אט בזמן אמת.

כללי יסוד למימוש

  1. שכבת אורקסטרציה, לא מערכת חדשה — חוזרים על מנוע הדיוור, המועדון וה‑CRM הקיימים. אסור לשכפל רשימות, עורך מיילים או טבלת אנשי קשר.
  2. בלעדיות דרך סטטוס — הרשימות הן סגמנטים דינמיים מעל contacts.status. ליד שמשלם יוצא מהלידים ונכנס ללקוחות מאליו — ללא "העברה" ידנית, ללא כפל מסרים.
  3. אורקסטרציה אטומית ואידמפוטנטית — יצירת הוובינר מבצעת את כל ה‑provisioning בטרנזקציה אחת עם rollback; ריצה חוזרת לא יוצרת כפילויות.
  4. אירועים, לא polling — webinar_registered / payment.succeeded מניעים תיוג, מעבר סטטוס וגישת צ׳אט דרך תור אירועים ו‑Trigger.dev.
  5. חדר צ׳אט = לקוחות בלבד — הרשאת הצטרפות לערוץ מונפקת רק כש‑status='customer' AND has_tag(webinar). מקור אמת ב‑Postgres, שידור דרך שכבת pub/sub.

01סקירה ומבנה

המודול יושב תחת מרכז בקרה הנהלה. הוא מוסיף ישות אחת חדשה (webinars) ושלוש טבלאות צ׳אט, וקושר ביניהן לבין הטבלאות הקיימות של הדיוור וה‑CRM. כל היתר — תיוג, סגמנטים, סדרות, שליחות — הוא שימוש חוזר.

ניהול וובינרים

רשימת אירועים, סטטוס, ומשפך-מיני (נרשמו/שילמו/%) לכל וובינר.

יצירת וובינר

טופס + פאנל "מה ייווצר אוטומטית": 2 רשימות, חדר צ׳אט, שלד סדרה.

וובינר בודד

משפך מלא, רשימות מקושרות, סדרת מיילים מעוגנת-מועד, חדר צ׳אט.

רשימות דיוור קיים

מנוע הדיוור — סגמנטים, קהלים, קמפיינים, סדרות. נצרך כמו שהוא.

חדר צ׳אט מועדון קיים

המועדון — מקבל סקשן "וובינרים" עם ערוץ לקוחות-בלבד פר-וובינר.

סטטוס לקוח קיים

ה‑CRM/סליקה — contacts.status הוא ציר הבלעדיות.

מקור אמת יחיד: contacts ו‑tags משותפים עם ה‑CRM ומנוע הדיוור (ראו "מערכת דיוור הנהלה — מדריך יישום CTO"). מערכת הוובינרים אינה מנהלת אנשי קשר נפרדים — היא מתייגת ומסנפת מעל אותן טבלאות.

02ארכיטקטורת האורקסטרציה

מסע המשתמש חוצה שלושה מודולים. הוובינר הוא ה‑hub שמחבר ביניהם דרך תגית אחת (webinar:<slug>) וסטטוס איש הקשר.

הרשמה
נרשם לוובינר
דף הרשמה פולט webinar_registered → תיוג webinar:<slug>, סטטוס נשאר lead.
←
רשימת לידים
סגמנט לידים
status='lead' AND tag — נכנס לסדרת המיילים המתוזמנת לוובינר.
←
המרה
משלם → לקוח
payment.succeeded → status='customer'. יוצא מהלידים מאליו.
←
מועדון + צ׳אט
חבר מועדון
מצורף למועדון + chat_channel_members → גישה לחדר הוובינר.

נקודת המפתח: מעבר הסטטוס הוא אירוע יחיד שמשנה בו-זמנית את שלוש התוצאות — יציאה מסגמנט הלידים, כניסה לסגמנט הלקוחות, וגישה לצ׳אט. אין שלוש פעולות שעלולות להתנתק זו מזו.

03מודל הנתונים (DB Schema)

חדש: webinars + 3 טבלאות צ׳אט + הרחבת sequence_steps. כל השאר (contacts, tags, tag_rules, segments, audiences, sequences, email_docs, email_sends) קיים ממנוע הדיוור.

-- ── ⭐ טמפלייט וובינר — בלופרינט שמשוכפל לכל וובינר חדש ──
CREATE TABLE webinar_templates (
  id             uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name           text NOT NULL,
  blueprint_json jsonb NOT NULL,   -- סדרה, צ׳אט, נחיתה, הצעה+מסלולים
  is_canonical   boolean NOT NULL DEFAULT true,  -- הגרסה המנצחת/הפעילה
  created_at     timestamptz NOT NULL DEFAULT now()
);

-- ── ישות הוובינר (ה-hub) ──
CREATE TABLE webinars (
  id                   uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name                 text NOT NULL,
  slug                 text NOT NULL UNIQUE,
  description          text,
  scheduled_at         timestamptz NOT NULL,
  status               text NOT NULL DEFAULT 'draft',  -- draft|published|live|ended
  host_name            text,
  cover_image_url      text,                  -- Bunny CDN
  registration_url     text,
  video_asset_id       text,                  -- Bunny Stream (לייב/הקלטה + retention)
  template_id          uuid REFERENCES webinar_templates(id), -- מאיזה טמפלייט שוכפל
  variant_group        text,                  -- מזהה קבוצת A/B (וריאנטים חולקים)
  variant_label        text,                  -- 'A' | 'B' | ...
  -- קישורי האורקסטרציה (נוצרים אוטומטית ביצירה):
  tag_id               uuid REFERENCES tags(id),
  lead_audience_id     uuid REFERENCES audiences(id),
  customer_audience_id uuid REFERENCES audiences(id),
  chat_channel_id      uuid REFERENCES chat_channels(id),
  sequence_id          uuid REFERENCES sequences(id),
  created_at           timestamptz NOT NULL DEFAULT now(),
  updated_at           timestamptz NOT NULL DEFAULT now()
);

-- ── צ׳אט המועדון (מקור אמת; השידור החי מעל זה) ──
CREATE TABLE chat_channels (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name        text NOT NULL,
  section     text NOT NULL,            -- 'general' | 'courses' | 'webinars'
  visibility  text NOT NULL DEFAULT 'all',  -- 'all' | 'customers'
  webinar_id  uuid REFERENCES webinars(id),
  created_at  timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE chat_channel_members (
  channel_id uuid REFERENCES chat_channels(id) ON DELETE CASCADE,
  contact_id uuid REFERENCES contacts(id) ON DELETE CASCADE,
  joined_at  timestamptz NOT NULL DEFAULT now(),
  PRIMARY KEY (channel_id, contact_id)
);

CREATE TABLE chat_messages (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  channel_id  uuid REFERENCES chat_channels(id) ON DELETE CASCADE,
  contact_id  uuid REFERENCES contacts(id),
  body        text NOT NULL,
  created_at  timestamptz NOT NULL DEFAULT now()
);

-- ── Retention: חלון צפייה לכל צופה → עקומת צופים בו-זמנית ──
CREATE TABLE webinar_attendance (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  webinar_id    uuid REFERENCES webinars(id) ON DELETE CASCADE,
  contact_id    uuid REFERENCES contacts(id),
  joined_at     timestamptz NOT NULL,
  left_at       timestamptz,
  watch_seconds int
);

-- ── אירועי עגלה והמרה (נכנסו/נטשו/לקוח + מסלול) ──
CREATE TABLE webinar_conversions (
  id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  webinar_id  uuid REFERENCES webinars(id) ON DELETE CASCADE,
  contact_id  uuid REFERENCES contacts(id),
  stage       text NOT NULL,    -- 'checkout_started'|'checkout_abandoned'|'purchased'
  plan        text,             -- 'teacher'|'educator'|'leader'|'enterprise' (מסלול WHITEBOARD)
  amount      numeric,
  created_at  timestamptz NOT NULL DEFAULT now()
);

-- ── הרחבת מנוע הסדרות: עיגון צעד למועד הוובינר ──
ALTER TABLE sequence_steps
  ADD COLUMN anchor        text DEFAULT 'enrollment',  -- 'enrollment' | 'webinar_start'
  ADD COLUMN offset_amount int,
  ADD COLUMN offset_unit   text;                       -- 'hours' | 'days'  (שלילי = לפני)

Prisma: שמות מודלים PascalCase (Webinar, ChatChannel, ChatMessage), שדות camelCase (scheduledAt, coverImageUrl), enums UPPER_SNAKE (DRAFT, PUBLISHED, LIVE, ENDED). תמיד createdAt+updatedAt.

04מנגנון הבלעדיות (סגמנטים דינמיים)

הדרישה: ליד שהפך ללקוח מפסיק מיידית לקבל מסרים מרשימת הלידים. הפתרון אינו "להעביר" אותו בין רשימות — אלא להגדיר את שתי הרשימות כ‑סגמנטים דינמיים שתלויים ב‑status. החברות מחושבת מחדש בכל שליחה, ולכן המעבר אוטומטי לחלוטין.

// סגמנט "לידים - הורות רגועה" — נשלף בזמן שליחה
{
  "logic": "and",
  "conditions": [
    { "field": "status", "op": "eq",  "value": "lead" },
    { "field": "tag",    "op": "has", "value": "webinar:horut-reguah" }
  ]
}

// סגמנט "לקוחות - הורות רגועה" — אותו tag, status שונה
{
  "logic": "and",
  "conditions": [
    { "field": "status", "op": "eq",  "value": "customer" },
    { "field": "tag",    "op": "has", "value": "webinar:horut-reguah" }
  ]
}

// מעבר סטטוס יחיד → שינוי חברות בשני הסגמנטים בו-זמנית:
UPDATE contacts SET status = 'customer' WHERE id = :contactId;
// מעכשיו: נופל מהסגמנט 'lead' ונכלל בסגמנט 'customer'. אין מסר נוסף כליד.

למה לא רשימות סטטיות: רשימה סטטית (snapshot) הייתה דורשת job שמסיר ידנית את הלקוח מרשימת הלידים — נקודת כשל (race / מסר שיוצא בין ההמרה להסרה). סגמנט דינמי מבטל את הבעיה בשורש: אין מצב ביניים.

סדרה פעילה: אם ליד נמצא באמצע סדרת מיילים ומומר ללקוח, יש לסיים את ההרשמה לסדרת הלידים (sequence_enrollments.status='exited') ביציאתו מהסגמנט — אחרת צעדים שנותרו עלולים להישלח. ה‑exit מופעל מאותו אירוע payment.succeeded.

05אוטומציית היצירה (Trigger.dev)

שמירת וובינר חדש מפעילה עבודת provision-webinar — טרנזקציה אטומית ואידמפוטנטית שיוצרת את כל הקישורים. כשל בכל שלב מגלגל אחורה את כולם, כך שלעולם אין "וובינר בלי חדר" או "רשימה בלי כלל תיוג".

// trigger/webinar/provision-webinar.ts
export const provisionWebinar = task({
  id: "provision-webinar",
  run: async (p: { webinarId: string }) => {
    await db.transaction(async (tx) => {
      const w = await getWebinar(tx, p.webinarId);
      const slug = w.slug;                       // webinar:<slug>

      // 1) תגית + כלל תיוג אוטומטי (נרשם → מתויג)
      const tag = await upsertTag(tx, { name: `webinar:${slug}`, kind: "auto" });
      await upsertTagRule(tx, {
        triggerEvent: "webinar_registered",
        conditionsJson: { webinarId: w.id },
        tagId: tag.id,
      });

      // 2) שני סגמנטים דינמיים + שני קהלים
      const leadSeg = await upsertSegment(tx, segDef("lead", slug));
      const custSeg = await upsertSegment(tx, segDef("customer", slug));
      const leadAud = await upsertAudience(tx, { name: `לידים - ${w.name}`, type: "segment", segmentId: leadSeg.id });
      const custAud = await upsertAudience(tx, { name: `לקוחות - ${w.name}`, type: "segment", segmentId: custSeg.id });

      // 3) חדר צ׳אט במועדון (לקוחות בלבד)
      const channel = await upsertChannel(tx, {
        name: w.name, section: "webinars", visibility: "customers", webinarId: w.id,
      });

      // 4) שלד סדרת מיילים (טריגר: נרשם לוובינר)
      const seq = await upsertSequence(tx, {
        name: `סדרת וובינר - ${w.name}`,
        triggerEvent: "webinar_registered",
        triggerConfigJson: { webinarId: w.id },
      });

      // 5) קישור חזרה ל-webinars (אידמפוטנטי על ה-IDs)
      await linkWebinar(tx, w.id, {
        tagId: tag.id, leadAudienceId: leadAud.id, customerAudienceId: custAud.id,
        chatChannelId: channel.id, sequenceId: seq.id,
      });
    });
  },
});

// segDef("lead","horut-reguah") → conditions_json של הסגמנט
function segDef(status: string, slug: string) {
  return { name: `${status}:${slug}`, isDynamic: true, logic: "and",
    conditionsJson: { conditions: [
      { field: "status", op: "eq",  value: status },
      { field: "tag",    op: "has", value: `webinar:${slug}` },
    ] } };
}

אידמפוטנטיות: כל ה‑upsert* מבוססי מפתח דטרמיניסטי (ה‑slug / webinarId). הרצה חוזרת של ה‑provision (deploy / retry) לא יוצרת כפילויות. שינוי שם וובינר מריץ rename-webinar-links שמעדכן שמות תצוגה בלבד — ה‑slug וה‑IDs קבועים.

06נתונים: משפך, Retention, הכנסה

מסך הוובינר ממוקד בדאטא ובמסקנות (לא בניהול דיוור). המדדים נגזרים מהתגית מול ה‑status; מעליהם נבנים משפך תוצאות מלא, עקומת Retention על הסרטון, וסיכום הכנסה — כולם נתונים מדויקים מהמערכת (ללא מדדי עלות/PPC).

-- משפך וובינר בודד
SELECT
  count(*)                                            AS registered,    -- נרשמו
  count(*) FILTER (WHERE c.status = 'lead')        AS still_lead,    -- עדיין ליד
  count(*) FILTER (WHERE c.status = 'customer')    AS converted,     -- שילמו
  round(100.0 * count(*) FILTER (WHERE c.status='customer')
        / nullif(count(*),0), 1)               AS conversion_pct
FROM contacts c
JOIN contact_tags ct ON ct.contact_id = c.id
JOIN tags t          ON t.id = ct.tag_id
WHERE t.name = 'webinar:horut-reguah';

משפך תוצאות מלא + מסלול לקוח

-- הרשמה → צפייה → עגלה → נטישה → לקוח, ופילוח לפי מסלול
SELECT
  (SELECT count(*) FROM webinar_attendance     WHERE webinar_id = :w)                          AS viewed,
  (SELECT count(*) FROM webinar_conversions wc WHERE wc.webinar_id=:w AND stage='checkout_started')   AS cart,
  (SELECT count(*) FROM webinar_conversions wc WHERE wc.webinar_id=:w AND stage='checkout_abandoned') AS abandoned,
  (SELECT count(*) FROM webinar_conversions wc WHERE wc.webinar_id=:w AND stage='purchased')          AS customers;

-- לקוחות והכנסה לפי מסלול
SELECT plan, count(*) AS customers, sum(amount) AS revenue
FROM webinar_conversions
WHERE webinar_id = :w AND stage = 'purchased'
GROUP BY plan;

Retention — עקומת צופים בו-זמנית

כל שורת נוכחות היא חלון [joined_at, left_at]. כדי לבנות עקומה לאורך דקות הוובינר, דוגמים את מספר הצופים הפעילים בכל "דקה" (bucket) — בסיס לגרף ה‑retention ולנקודות הצניחה.

-- מספר צופים פעילים בכל דקה מתחילת הוובינר
SELECT m.minute,
       count(a.*) AS concurrent_viewers
FROM generate_series(0, 60) AS m(minute)
LEFT JOIN webinar_attendance a
  ON a.webinar_id = :w
 AND a.joined_at <= :start + (m.minute * interval '1 min')
 AND coalesce(a.left_at, now()) >= :start + (m.minute * interval '1 min')
GROUP BY m.minute ORDER BY m.minute;

סיכום הכנסה

-- עובדות הכנסה בלבד (ללא עלות/ROI) מתוך רכישות אמיתיות
SELECT
  sum(amount)                              AS mrr,          -- הכנסה חודשית מתחדשת
  sum(amount) * 10                         AS annual_value, -- שווי שנתי (שלם 10 קבל 12)
  count(*)                                 AS customers,
  round(sum(amount) / nullif(count(*),0))   AS avg_per_customer
FROM webinar_conversions
WHERE webinar_id = :w AND stage = 'purchased';

מקורות: webinar_attendance מוזן מהלייב/נגן הוידאו (heartbeat), ו‑webinar_conversions מהסליקה/עגלה. שאילתות אלו מזינות ישירות את לשונית "נתונים ודאטא" ואת "מסקנות".

הוסר במכוון: מדדי PPC/ROI (עלות מדיה, CPL, CPA, החזר השקעה) ירדו מהמערכת — הם דרשו אינטגרציית Meta/Google מורכבת ושיוך לא-מדויק. המסך מתמקד בנתונים מדויקים: צפייה, המרה והכנסה.

06bטמפלייט וובינר + שכפול + A/B

עיקרון מרכזי: מגדירים את הפלואו פעם אחת כ-טמפלייט (webinar_templates.blueprint_json), ומשכפלים אותו לכל וובינר חדש במקום לבנות מאפס. ה-blueprint מכיל את סדרת המיילים, הגדרות הצ׳אט/נחיתה, וההצעה והמסלולים.

// trigger/webinar/clone-from-template.ts — יצירת וובינר/וריאנט מטמפלייט
export const cloneFromTemplate = task({
  id: "clone-from-template",
  run: async (p: { templateId: string, name: string, scheduledAt: string,
                  variantGroup?: string, variantLabel?: string }) => {
    const tpl = await getTemplate(p.templateId);
    const w   = await createWebinar({                 // מאוכלס מה-blueprint
      name: p.name, scheduledAt: p.scheduledAt, templateId: tpl.id,
      variantGroup: p.variantGroup, variantLabel: p.variantLabel,
      ...materialize(tpl.blueprintJson),            // סדרה, צ׳אט, נחיתה, הצעה
    });
    await provisionWebinar.trigger({ webinarId: w.id }); // תגית/סגמנטים/חדר/סדרה משלו
    return w;
  },
});

A/B Test: שני וובינרים עם אותו variant_group ו-variant_label שונה הם וריאנטים. התנועה מתפצלת, ומדד ההכרעה הוא אחוז ההמרה מאותו משפך (סעיף 06). הוריאנט המנצח מקודם ל-is_canonical=true בטמפלייט — וכל וובינר עתידי משוכפל ממנו. כך ההצלחה מצטברת.

"שכפל את ההצלחה": אפשר גם לשכפל וובינר קיים מצליח ישירות לטמפלייט חדש (is_canonical) — בלי לבנות פלואו מאפס. השכפול הוא של המבנה בלבד; הדאטא (לידים/הודעות/שליחות) של כל וובינר נשאר נפרד.

07סדרת מיילים מעוגנת-מועד

מנוע הסדרות הקיים מקדם הרשמות לפי next_run_at. ההרחבה: כש‑step.anchor='webinar_start', ה‑next_run_at מחושב מול webinar.scheduled_at ± offset במקום מול זמן ההרשמה. כך אפשר "יום לפני", "שעה לפני", "שעתיים אחרי" — כולם יחסית למועד האירוע, גם אם נרשמים בזמנים שונים.

// חישוב מועד צעד בעת קידום הרשמה (advance-enrollments מורחב)
function computeNextRun(step: SequenceStep, enr: Enrollment, webinar: Webinar) {
  if (step.anchor === "webinar_start") {
    return addTime(webinar.scheduledAt, step.offsetAmount, step.offsetUnit);
    // offsetAmount שלילי = לפני הוובינר (-1 day / -1 hour)
  }
  return addTime(enr.enrolledAt, step.waitAmount, step.waitUnit); // התנהגות קיימת
}

// צעד תנאי: הסתעפות לפי משתנה נרשם / לא-נרשם (ובהמשך הגיע/לא-הגיע)
function evalBranch(step: SequenceStep, c: Contact): "yes" | "no" {
  switch (step.conditionJson.variable) {
    case "registered": return hasTag(c, step.conditionJson.webinarTag) ? "yes" : "no";
    case "attended":   return hasTag(c, "attended:" + step.conditionJson.slug) ? "yes" : "no";
  }
}

מקור האמת להודעות הוא Postgres (chat_messages). השידור החי נעשה מעל שכבת pub/sub — Ably/Pusher כ‑MVP, או LiveKit data-channels הקיים בסטאק. ערוץ פר-וובינר: webinar:<id>.

שרשרת הרשאות (customers-only)

// app/api/chat/[channelId]/token/route.ts — הנפקת טוקן הצטרפות
export async function POST(req, { params }) {
  const contact = await requireSession(req);
  const ch = await getChannel(params.channelId);

  // חדר וובינר = לקוחות בלבד עם תגית הוובינר
  if (ch.visibility === "customers") {
    const ok = contact.status === "customer"
            && await hasTag(contact.id, `webinar:${ch.webinarSlug}`);
    if (!ok) return json({ error: "forbidden", code: "NOT_CUSTOMER" }, 403);
  }
  return json({ token: issueRealtimeToken(contact.id, `webinar:${ch.id}`) });
}

// שליחת הודעה: כותבים ל-Postgres (מקור אמת) ואז משדרים
async function sendMessage(channelId, contactId, body) {
  const msg = await db.chatMessage.create({ data: { channelId, contactId, body } });
  await realtime.publish(`webinar:${channelId}`, "message", serialize(msg));
  return msg;
}
